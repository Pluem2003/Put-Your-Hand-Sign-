## 1. ที่มาและแนวคิดของโปรเจค
โปรเจคนี้เริ่มต้นจากการที่เราอยากลองทำเกมที่ใช้เงามือเป็นคอนเซปหลัก โดยตั้งชื่อว่า **"Put Your Hand Sign!"** แนวคิดคือ การทำมือให้เกิดเงาแบบต่างๆ ตามโจทย์ที่ปรากฏบนหน้าจอ โดยมีเป้าหมายเพื่อให้ระบบ Computer Vision ตรวจจับและตรวจความถูกต้องของเงาได้รวดเร็ว จนสามารถนำมาแข่งกันได้ว่าใครทำตามโจทย์ได้เร็วกว่ากัน

> **[IMAGE 01: รูปภาพจำลองส่วน Player Input หรือการตั้ง Shadow Hand Station สำหรับสร้างเงามือ]**

## 2. รูปแบบการเล่นและกติกา
ตัวเกมถูกออกแบบมาให้เป็นการแข่งขันระหว่างผู้เล่น 2 ฝ่าย โดยมีหัวใจหลักคือการสร้างเงามือให้ถูกต้องและรวดเร็วที่สุด:

*   **ฝ่ายตั้งโจทย์ (The Challenger):** สามารถเลือกท่าทางเงามือจากระบบ หรือวางแผนเลือกท่าทางที่ยากเพื่อส่งเป็นโจทย์ไปท้าทายอีกฝ่าย รวมถึงสามารถ "สั่งห้ามทำท่าทาง" เพื่อสร้างความสับสนให้คู่ต่อสู้ได้
*   **ฝ่ายทำตาม (The Player):** ต้องยื่นมือเข้าสู่จุดรับสัญญาณเพื่อสร้างเงามือเลียนแบบโจทย์ที่เห็นบนหน้าจอให้ทันเวลา โดยมีเวลากดดันเพียงท่าละ 7 วินาที
*   **การตัดสิน:** ในหนึ่งรอบจะมีโจทย์เงามือต่อเนื่อง 5 ท่า หากทำผิดเงื่อนไขครบ 3 ครั้ง ระบบจะปรับแพ้ทันที (Game Over) โดยผู้เล่นจะได้ 1 คะแนนต่อท่าที่ทำถูกต้อง

## 3. ขั้นตอนการดำเนินงานอย่างละเอียด (Methodology)

ในการพัฒนาระบบนี้ เราแบ่งขั้นตอนการทำงานออกเป็น 3 ส่วนหลัก เพื่อให้ได้ระบบที่เสถียรและแม่นยำที่สุด:

### 3.1 การเก็บข้อมูลและการประมวลผลเบื้องต้น (Data Collection & Pre-processing)
อุปสรรคที่ใหญ่ที่สุดของการทำเงามือคือ "สภาพแสง" ที่ไม่คงที่ เราจึงใช้เทคนิคการประมวลผลภาพผ่าน **OpenCV** ตั้งแต่ขั้นตอนการเก็บ Dataset โดยใช้การทำ **Binary Thresholding** เพื่อแปลงภาพจากกล้องให้เป็นสีขาว-ดำทันที

> **[IMAGE 02: ภาพเปรียบเทียบ Original vs Binary Thresholding ในสภาวะแสงต่างๆ]**

*   **การควบคุมสภาพแสง:** การแปลงเป็น Binary ช่วยให้เราตัดสัญญาณรบกวนจากสีผิว พื้นหลัง และเงาสะท้อนที่ไม่จำเป็นออกไป ทำให้ AI โฟกัสเฉพาะ "โครงร่าง" (Contour) ของมือเท่านั้น
*   **Dataset Classes:** เราทำการรวบรวมข้อมูลทั้งหมด 10 Classes โดยแบ่งเป็นสัตว์ 7 ชนิด และท่าค้อนกรรไกรกระดาษ 3 ท่า

> **[IMAGE 03: แกลเลอรีภาพตัวอย่างของทั้ง 10 คลาส (Bird, Bull, Crab, Deer, Dog, Duck, Elephant, Paper, Rock, Scissor)]**

### 3.2 การฝึกสอนโมเดล (Model Training)
เราเลือกใช้โมเดล **YOLO26** ซึ่งเป็นสถาปัตยกรรมที่โดดเด่นเรื่องความเร็วในการประมวลผล โดยใช้ **Google Colab** ในการฝึกสอนโมเดล

> **[IMAGE 04: ภาพหน้าจอการ Training บน Google Colab และกราฟแสดง Loss/Accuracy]**

โค้ดส่วนสำคัญในการประมวลผลภาพก่อนส่งให้ AI (Inference Logic):
```python
# ตัวอย่างโค้ดการทำ Thresholding และเตรียมภาพเข้า YOLO26
gray = cv2.cvtColor(current_raw_frame, cv2.COLOR_BGR2GRAY)
_, binary = cv2.threshold(gray, THRESHOLD_VAL, 255, cv2.THRESH_BINARY)
input_img = cv2.resize(binary, (320, 320))
input_img = cv2.cvtColor(input_img, cv2.COLOR_GRAY2BGR)

# ทำนายผลด้วยโมเดล YOLO26
results = model.predict(source=input_img, conf=0.5, verbose=False, imgsz=320)
```

### 3.3 การพัฒนาส่วนแสดงผลและการสื่อสาร (Web Interface & Communication)
เราใช้ **Node.js** ร่วมกับ **Socket.io** เป็นตัวกลางในการสื่อสารระหว่างส่วนประมวลผล AI (Python) และส่วนแสดงผลหน้าเว็บ (Browser)

> **[IMAGE 05: หน้าจอ UI ฝั่ง Spectator แสดงการแข่งระหว่าง Player 1 และ Player 2]**

โค้ดส่วนการรับส่งข้อมูลผ่าน Socket.io จาก Python ไปยัง Server:
```python
# การส่งผลลัพธ์การทำนายและภาพ Binary ไปยัง Server
payload = {
    'playerId': PLAYER_ID,
    'prediction': latest_prediction,
    'confidence': latest_confidence,
    'cameraFrame': buffer.tobytes(), # ส่งภาพในรูปแบบ Binary
    'inferenceTime': latest_inference_time,
    'timestamp': time.time()
}
sio.emit('player_pose', payload)
```

หน้าจอสำหรับจัดการระบบ (Debug Console) ช่วยให้เราทดสอบระบบ Logic ได้ง่ายขึ้น:
> **[IMAGE 06: ภาพหน้าจอ Debug Console แสดงปุ่มควบคุมเกมและสถานะแบบ Real-time]**

## 4. ผลการทดสอบและการวัดผล (Evaluation)

จากการทดสอบระบบในสภาพแวดล้อมจริง เราวัดผลจากความแม่นยำและการตอบสนองของระบบ:
*   **Accuracy:** โมเดลมีความแม่นยำสูงแม้เงามือจะมีการขยับเล็กน้อย
*   **Inference Speed:** ความเร็วในการประมวลผลอยู่ที่ประมาณ 10-20 ms ต่อเฟรม ซึ่งถือว่าเพียงพอสำหรับการเล่นเกมแข่งขัน
